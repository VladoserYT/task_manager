# Проект: Менеджер задач (Task Manager)

## 1. Введение
Настоящий проект представляет собой desktop-приложение "Менеджер задач", разработанное в рамках выполнения лабораторной работы по дисциплине "Программирование". Целью проекта является демонстрация навыков объектно-ориентированного программирования (ООП), работы с графическим интерфейсом пользователя (GUI) и реализации механизмов персистентного хранения данных.

## 2. Технологический стек
Проект реализован на языке программирования **Python 3.x** с использованием следующих ключевых компонентов:
*   **Tkinter:** Стандартная библиотека Python для создания графического интерфейса пользователя (GUI).
*   **JSON (JavaScript Object Notation):** Используется для структурированного хранения данных о задачах во внешнем файле.
*   **ООП (Объектно-ориентированное программирование):** Приложение построено на основе системы классов для инкапсуляции логики и данных.

## 3. Архитектура и принцип работы
Приложение имеет двухкомпонентную структуру, реализованную через систему классов:

### 3.1. Модуль `classes.py` (Бизнес-логика)
Модуль содержит два основных класса, реализующих логику управления задачами:

| Класс | Назначение | Основные методы |
| :--- | :--- | :--- |
| **`Task`** | Объектное представление отдельной задачи. Инкапсулирует данные о задаче. | `__init__(title, description, due_date)` |
| **`TaskManager`** | Управляет коллекцией объектов `Task`. Отвечает за файловые операции и манипуляции со списком задач. | `add_task()`, `delete_task()`, `save_to_file()`, `load_from_file()` |

**Принцип персистентности:** Класс `TaskManager` автоматически вызывает метод `load_from_file()` при инициализации для загрузки ранее сохраненных данных из `tasks.json` и метод `save_to_file()` после каждой операции добавления или удаления задачи, обеспечивая актуальность данных.

### 3.2. Модуль `gui.py` (Графический интерфейс)
Модуль содержит класс `TaskManagerApp`, который отвечает за построение и управление элементами GUI с использованием Tkinter.
*   **Взаимодействие:** Класс `TaskManagerApp` взаимодействует с экземпляром `TaskManager` для выполнения операций (добавление, удаление) на основе действий пользователя в интерфейсе.
*   **Виджеты:** Используются стандартные виджеты Tkinter (`Entry`, `Text`, `Button`, `Listbox`) для ввода данных и отображения списка задач.

## 4. Функциональные возможности
Приложение обеспечивает следующий функционал:
1.  **Добавление задачи:** Ввод названия, описания и срока выполнения с последующим сохранением в списке и файле.
2.  **Удаление задачи:** Удаление выбранного элемента из списка с автоматическим обновлением файла хранения.
3.  **Автоматическое сохранение/загрузка:** Обеспечение персистентности данных через файл `tasks.json`.

## 5. Инструкция по запуску
Для запуска приложения необходимо наличие интерпретатора Python 3.x.

1.  **Размещение файлов:** Поместить файлы `classes.py` и `gui.py` в один рабочий каталог.
2.  **Запуск:** В командной строке (терминале) перейти в рабочий каталог и выполнить команду:
    ```bash
    python gui.py
    ```
    *(В случае проблем с командой `python`, использовать `python3`)*

3.  **Результат:** При успешном запуске будет отображено окно графического интерфейса "Менеджер задач".
